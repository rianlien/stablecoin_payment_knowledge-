# h402

## Overview

h402 は bit-gpt が公開している HTTP 402 payments project。x402 の open scheme を基礎にしつつ、EVM、Solana、Arkade、broadcasted transactions まで扱い、`upto`、`streamed`、`subscription` などの payment model 拡張も roadmap に含める。

## User Goal

HTTP-native paywall / agent payments を、複数 network と異なる settlement model で比較しながら実装する。

## Business Goal

x402 互換圏で別実装を提示し、builder に network coverage と payment model の選択肢を出す。

## Category

- x402
- agent-payments

## Core Workflow

1. Developer は monorepo から package と example app を取得する。
2. Next.js middleware などで payment-gated route を設定する。
3. Client は HTTP 402 応答に従って native blockchain transaction を送る。
4. Facilitator / server が支払い確認後に resource access を返す。

GitHub README では未確認。OSS distribution と将来的な hosted service 余地がある。

## Differentiation

- x402 の open scheme を引き継ぎつつ、独自 implementation として network / payment type を広げている。
- USDT のような broadcasted transaction と permit token の両方を扱う。
- roadmap で subscription / streamed payments まで見せている。

## Operational Notes

- README で `small team` による rapid iteration を明言しているため、仕様安定性は要確認。
- example app と facilitator を含むので比較検証用としては使いやすい。
- canonical x402 との差分が interoperability にどう影響するかを見る必要がある。

## Risks / Open Questions

- spec divergence が大きいと ecosystem fragmentation を招く。
- roadmap が豊富でも merchant-side ops artifact が未整備だと導入は進みにくい。
- Bitcoin 系を含む network expansion が本当に merchant value に繋がるかは未検証。
