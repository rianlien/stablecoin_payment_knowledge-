# T402

## Overview

T402 は、USDT を中心にした HTTP-native payment protocol と SDK 群。API monetization、AI agent payments、content paywall を multi-chain で実装できるようにする。

## User Goal

developer が stablecoin payment を HTTP request / response に直接組み込み、wallet や chain 差分を最小化して実装したい。

## Business Goal

T402 が USDT・multi-chain 前提の payment protocol / tooling layer として ecosystem を取ること。

## Category

x402, agent-payments

## Core Workflow

server は保護対象 endpoint に payment requirement を返し、client は署名ベースの支払いを行って resource access を得る。SDK は server middleware、wallet abstraction、bridge、multisig、MCP 連携まで持つ。

## Pricing / Business Model

open-source protocol / SDK 型。商用 monetization は facilitator、enterprise support、周辺サービスに寄る可能性があるが source 上で未確認。

## Differentiation

USDT、47 networks、wallet abstraction、bridge、multisig を一つの product surface として見せており、USDC / Base 寄り実装との差が明確。

## Operational Notes

導入前に facilitator trust model、refund / dispute semantics、merchant accounting impact、supported production networks、security review を確認したい。

## Risks / Open Questions

- commercial support と governance model が未確認
- multi-chain breadth が merchant-side ops を複雑化する恐れ
- usage dispute と fulfillment failure の標準処理が見えにくい
