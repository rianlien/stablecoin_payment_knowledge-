## 概要
- dLocal は 2026 年 4 月 21 日に「Stablecoin Full」ソリューションを正式発表
- 対象は dLocal がすでにカバーする 44 か国以上の新興国市場（ラテンアメリカ・アフリカ・アジアが中心）
- 加盟店は収納（pay-in）・ペイアウト（pay-out）・財務管理（treasury）をステーブルコインで単一 API から実行可能
- FX 換算・照合・レポーティング機能も統合済み

## 何が起きたか
- **主な機能**：
  1. ステーブルコインでの直接決済受付（チェックアウト時）
  2. 決済金の精算を USD またはステーブルコインで選択可能
  3. グローバルへのステーブルコインペイアウト（クリエイターペイ・サプライヤー払い等）
  4. 現地通貨とステーブルコインの相互換算
  5. レポーティング・照合機能（ひとつの統合に内包）
- dLocal の既存のライセンス・コンプライアンスネットワーク（各国ごと）を通じてステーブルコインフローを処理し、現地の規制・データ要件に準拠させる設計
- ステーブルコインを既存の現地フィアットレールの「並列レール」として位置づけ、追加の暗号資産レイヤーではなく標準的な決済手段として扱う

## なぜ重要か

### 決済事業者視点
- 新興国向けの多通貨決済における FX ボラティリティ・送金コスト・流動性分散の問題に対し、PSP レベルでステーブルコインが制度的な解答を提示した事例
- 「ステーブルコイン対応 PSP」から「ステーブルコインをネイティブに持つ PSP」へのシフトを示している

### 加盟店視点
- 44 か国のローカル決済手段と同一の API でステーブルコインを受け入れられるため、導入摩擦が従来の暗号資産対応と比べて大幅に低下
- 精算通貨の選択肢（USD / ステーブルコイン）が財務戦略の柔軟性を高める

### プロダクト視点
- 収納・ペイアウト・財務管理の三機能統合は、ステーブルコイン決済基盤が「送金ツール」から「フルスタック PSP 機能」へ進化したことを示す
- 同様の三機能統合を自社プロダクトで検討する場合の参照モデルになりうる

### 規制 / リスク視点
- 各国ごとにライセンスを持つ dLocal のネットワークを通じてステーブルコインを処理するため、VASP 規制と PSP 規制の二重適用リスクを dLocal 側で吸収している設計
- ただし、各国の規制当局がステーブルコインを「電子マネー」として扱うか「暗号資産」として扱うかは依然として不確定要素

## 我々への示唆
- いま検討すべきこと:
  - 新興国市場でのステーブルコイン収納・送金ニーズがある加盟店を持つ場合、dLocal 連携を選択肢として評価する
  - 三機能統合モデル（収納・ペイアウト・財務管理）が自社プロダクトロードマップに示唆する要件を整理する
- 後で深掘りすべきこと:
  - 実際に対応しているステーブルコインの種類（USDC / USDT / その他）
  - 各国別の規制ライセンス状況と取扱可能通貨ペアの詳細
- 今は優先度が低いこと:
  - dLocal 固有の API 仕様（自社で新興国向け直接接続を計画しない限り）

## 未解決の論点
- 対応ステーブルコインは USDC / USDT 限定か、その他トークンも含むか
- 現地規制当局（例：ブラジル BCB・メキシコ CNBV）との事前調整はどの程度完了しているか
- FX 換算レートのソースと手数料体系が透明かどうか
