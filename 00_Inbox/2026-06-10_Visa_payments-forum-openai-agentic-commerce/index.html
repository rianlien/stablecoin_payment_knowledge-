    Visa、OpenAI と戦略的連携——AI エージェント向け「Visa Intelligent Commerce」・Agent Score・Agentic Registry・ステーブルコイン決済拡張を発表（Visa Payments Forum 2026）

    概要¶

    Visa が 2026 年 6 月 10 日、サンフランシスコで開催した Visa Payments Forum 2026 にて、AI エージェント時代に向けた複数の重要イニシアチブを発表。最大の注目は OpenAI との戦略的連携で、OpenAI のエージェント体験に Visa の決済インフラを統合することで、AI エージェントが Visa ネットワーク経由で安全に支払いを処理できるようにする。同日に Mastercard も AP4M をローンチしており、6 月 10 日は業界を通じた「エージェント決済インフラ集約」の象徴的な一日となった。

    何が起きたか¶

    1. Visa × OpenAI 戦略的連携¶

    • Visa が OpenAI との戦略的連携を発表
    • Visa はグローバルネットワーク・認証情報・セキュリティインフラを OpenAI のエージェント商取引体験に提供
    • 開発者・加盟店が OpenAI のエージェント機能内で Visa 決済を受け入れ可能になる
    • トランザクションは「ユーザー定義のパーミッションと管理（支出上限・加盟店カテゴリ・承認要件）」内で実行
    • OpenAI の Codex 技術を活用した企業向けワークフローの自動化も探索予定
    • 将来的には「AI が主要なデジタルインターフェースになる」自動化・会話型コマース体験への拡大を計画

    2. AI 機能の新発表¶

    • Agent Score：AI エージェントが加盟店ウェブサイトを「エージェントが検索・理解・タスク完了できるか」の観点でスコアリングする機能。New Generation（Visa 子会社）と共同開発。加盟店のエージェント対応度の標準評価指標になる可能性
    • Agentic Registry：AI エージェントの登録・検証機構（詳細は限定的）。エージェントの identity と認可を追跡するインフラ
    • Large Transaction Model：数十億件のトランザクションで学習した AI 不正検知モデル。不正検知の精度向上と誤検知（false decline）の削減を実現

    3. ステーブルコイン決済の拡大¶

    • VisaNet でのステーブルコイン決済：年換算約 35 億ドル（2026 年 3 月時点）
    • 発行体銀行（issuing bank）側ではすでに週 7 日のオンチェーン決済を実施
    • 新発表：acquirer 側にも週 7 日決済を拡大し、エコシステム全体の決済頻度・柔軟性を向上
    • ステーブルコイン決済パイロットを複数リージョン・チェーン・通貨に拡大予定

    4. 銀行向けデジタルマネー層¶

    • Visa が銀行の従来預金をプログラマブルなデジタルマネーに変換する技術層を構築中と表明
    • 「ステーブルコインの速度・柔軟性に匹敵しながら、資金を銀行バランスシート上に維持する」設計
    • GENIUS Act 施行後の銀行向けステーブルコイン発行（Fiserv FIUSD モデルに類似）との競合・補完関係

    確認された事実¶

    • 発表日・会場：2026-06-10・Visa Payments Forum（サンフランシスコ）
    • OpenAI 連携：Visa 公式プレスリリース + BusinessWire プレスリリースで確認
    • Agent Score：New Generation（Visa 子会社）との共同開発。加盟店のエージェント対応度評価ツール
    • Large Transaction Model：ビリオン単位トランザクションで学習した AI 不正検知モデル
    • ステーブルコイン決済量：年換算約 35 億ドル（2026 年 3 月時点、VisaNet 上）
    • acquirer 側週 7 日決済の拡大：今回の新発表
    • 銀行向けデジタルマネー層：「構築中」と表明
    • 出典：Visa 公式ニュースリリース・BusinessWire・Seeking Alpha・Fortune が同日報道

    なぜ重要か¶

    決済事業者視点¶

    • OpenAI は世界で最も利用されている AI プラットフォームの一つ。Visa がそのエージェント決済インフラに選ばれたことで、「消費者が AI エージェントに日常的な購入を委任する」シナリオで Visa レールが標準化するリスク・チャンスが生まれる
    • acquirer 側の週 7 日決済拡大は、エージェント決済の「24 時間・年中無休のリアルタイム決済」要件への対応。x402 / USDC ですでに 24/7 対応している競合インフラへの対抗措置として機能する

    加盟店視点¶

    • Agent Score によって「自分のサイトが AI エージェントに対応しているか」の客観的評価が得られる。エージェント対応改善の優先順位付けに使える実用ツールになりうる
    • OpenAI エージェントのユーザーが Visa 経由で支払えるようになることで、エージェント決済受け入れ加盟店として Visa 加盟店（全世界 1.3 億超）が自動的に対象になる可能性

    プロダクト視点¶

    • Agentic Registry のエージェント登録・検証仕様が公開された場合、AP2 の W3C VC ベースの認可設計や AP4M の Verifiable Intent との相互運用性が重要な設計判断点になる
    • 「ユーザー定義のパーミッション（支出上限・カテゴリ・承認要件）」はメタカード AP4M の Verifiable Intent と設計思想が類似——エージェント決済の「ユーザー最終管理」設計が業界横断でコンセンサス形成されつつある

    規制 / リスク視点¶

    • 銀行向けデジタルマネー層は、GENIUS Act の「PPSI（Permitted Payment Stablecoin Issuer）」モデルの外に位置するプログラマブル預金の可能性を示す。規制上の位置づけ（預金か電子マネーか）が GENIUS Act の適用範囲に影響する
    • Visa × OpenAI 連携の「ユーザー定義パーミッション」設計は、GENIUS Act の消費者保護要件および EU AI Act の人間監視義務との整合性が高い——規制当局が参照するユーザー管理モデルの先行事例になる可能性

    我々への示唆¶

    • いま検討すべきこと:
    • Agent Score のスコアリング基準が公開された際、自社プロダクトが「エージェント対応」と評価されるかどうかを評価する。x402 / AP2 対応の有無が Score に反映される場合、未対応は競争上の不利になる
    • Visa × OpenAI 連携の技術仕様（API・SDK）が公開された際、自社プロダクトが OpenAI エージェントを通じた Visa 決済を受け入れるために何が必要かを事前に把握する
    • 後で深掘りすべきこと:
    • Agentic Registry の登録要件——x402・AP2・AP4M との相互認証が可能かどうか（エージェント ID の一元化 vs 分散化）
    • VisaNet ステーブルコイン決済の対応チェーン・通貨詳細——USDC 以外の PPSI 認定ステーブルコインへの拡大タイムライン
    • 銀行向けデジタルマネー層の GENIUS Act 準拠設計——Circle/USDC のポジションにどう影響するか
    • 今は優先度が低いこと:
    • Visa Payments Forum の他の発表（Large Transaction Model の詳細——自社の不正検知設計に直接関係しない）

    ありそうな示唆¶

    • OpenAI が Mastercard（AP4M）にも参加するかどうか——現時点では Visa 独占連携。Mastercard も AI プラットフォームとの連携を発表する可能性があり、「エージェント決済の OpenAI 窓口」が Visa か Mastercard か両方かが次の観察点
    • Agent Score と Agentic Registry が業界標準の「エージェント対応度・信頼スコア」インフラになった場合、加盟店は Visa の評価基準に合わせたサイト設計・API 対応を優先するようになる——Visa によるエコシステム設計の主導権確保

    推測 / 監視ポイント¶

    • OpenAI エージェントが使える「Visa 決済」の製品範囲：Visa Debit か Credit か、仮想カードか、ステーブルコイン決済かによって、エージェント決済のコスト構造・加盟店採用動機が大きく異なる
    • 銀行向けデジタルマネー層のパートナー銀行の発表——JP Morgan・BofA・Wells Fargo 等の主要行が参加した場合、USDC の競合として市場インパクトが大きい
    • Visa の Agentic Registry がエージェント認証の「グローバル標準」として W3C・FIDO Alliance 等の標準化団体に提案されるかどうか

    未解決の論点¶

    • Visa × OpenAI 連携の「ユーザー定義パーミッション」の技術実装：エージェントが Visa ネットワークに直接アクセスするのか、OpenAI がブリッジとして機能するのか
    • ステーブルコイン 35 億ドルの「決済」の定義：カード決済のステーブルコイン精算か、オンチェーンダイレクト決済か
    • Agent Score の加盟店への開示・利用可能タイムライン
    • Agentic Registry と AP4M の Verifiable Intent の相互認識有無
