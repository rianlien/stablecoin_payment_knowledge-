# x402

x402 は HTTP 402 Payment Required を使い、API・コンテンツ・AI サービスをリクエスト単位で支払えるようにする agent / machine payments protocol。

会社としての Coinbase は [[Coinbase]]、標準化・ガバナンス組織としての x402 Foundation は [[x402 Foundation]]、エコシステム全体の俯瞰は [[x402-ecosystem]] を参照。

---

## 現状の要約

### 何を解くプロトコルか

- **API / AI / コンテンツの都度課金**: サブスクリプションや API key 発行なしに、リクエスト単位で支払いを要求・検証する。
- **AI エージェントによる自律決済**: 人間が都度 checkout しなくても、エージェントが HTTP レスポンスを読んで支払いを実行できる。
- **ステーブルコイン決済の埋め込み**: Base、Solana、Arbitrum などのチェーン上で USDC / ERC-20 等の支払いを行う文脈が多い。
- **マイクロペイメントの経済性改善**: バッチ決済により、サブセント単位の API / 推論 / データ課金を成立させる方向へ拡張している。

### 見立て

x402 は企業でもネットワークでもなく、支払いの要求・認証・実行を HTTP フローに埋め込むプロトコル。実装主体として Coinbase / CDP、実行環境として [[Base]]、標準化主体として [[x402 Foundation]] が関わるため、分析上はそれぞれを分けて扱う。

## 基本構造

| 要素 | 役割 | 確認ポイント |
|---|---|---|
| HTTP 402 response | 支払い条件を返す | API / コンテンツ提供者が価格・支払い先・通貨を提示する |
| Buyer / agent | 支払いを実行する主体 | AI エージェント、MCP クライアント、ウォレット連携アプリ |
| Facilitator | 支払い検証・処理を支援 | Coinbase CDP facilitator など。複数 facilitator 化が進むかが重要 |
| Settlement chain | 実際の決済レール | Base、Solana、Arbitrum 等 |
| Stablecoin / token | 支払い通貨 | USDC が中心。非 USD ステーブルコイン対応も拡大余地あり |

## 主要論点

### 承認ギャップ

x402 単体では支払い実行はできても、エージェントが人間の都度承認なしに支払い続けるための委任・支出制御レイヤーが別途必要になる。AP2 spending mandate、Fireblocks Policy Engine、ERC-7715 session key などとの関係が重要。

### 実需と投機の分離

Chainalysis などの分析では Base 上の x402 取引が累計 1 億件を超えたとされる一方、PING など投機的利用も含まれる。agentic commerce の実需を見るには、API、旅行、コンテンツ、推論課金などの用途別内訳が必要。

### マルチチェーン化

Base が中心だが、Solana、Arbitrum、XRPL、Casper、Injective などへの展開が進んでいる。マルチチェーン化は採用範囲を広げる一方、KYT、ステーブルコイン種別、facilitator 実装差分の複雑性を増やす。

### PSP / merchant readiness

加盟店にとっては「x402 endpoint を実装する」だけでは不十分で、返金、キャンセル、領収書、会計、Travel Rule、KYT、法定通貨精算との接続が必要になる。Fireblocks、AWS AgentCore、Coinbase CDP などの周辺インフラが普及を左右する。
