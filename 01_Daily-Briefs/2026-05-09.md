# Daily Brief: 2026-05-09

## 今日の要点

- Exodus Movement が 5 月 8 日、AI エージェント専用の Solana ステーブルコイン「XO Cash」と AgentKit SDK を正式ローンチ。MoonPay・Monavate との共同開発で、エージェントは秘密鍵を持たず Exodus Pay 残高から直接支出。ユーザーが 1 日上限・1 取引上限・利用可能加盟店・レート制限を設定でき、同設定は Visa 加盟店でのデビットカード決済にも自動適用される。USDC/USDT への自動変換にも対応。今週の MoonAgents Card（Monavate インフラ）と同一の基盤を活用した別ブランド製品であり、Monavate が複数の AI エージェント向けカード発行基盤として台頭しつつあることを示す
- OwlTing Group が 5 月 4 日、AI エージェント向けセルフカストディウォレット「OwlPay Agent Wallet」を正式ローンチ。米国 40 州の MTL と Visa Direct 連携（将来対応予定）を規制インフラとして保有し、ETH・Stellar・Solana の 3 チェーンに対応。組み込みエージェントスキルにより、エンドユーザーが技術設定なしでウォレット操作をエージェントに委任できる。AI エージェント決済インフラとして現時点で最も規制カバレッジが高い部類に入る

## 今日の重要論点

1. **エージェントウォレット群が多様化し「設計パターン」が収束しつつある**: 今週だけで AWS AgentCore・Pay.sh・MoonAgents Card・Exodus XO Cash・OwlPay Agent Wallet という 5 つの AI エージェント向け決済インフラが市場に登場した。共通して見られる設計パターンは「エージェントは秘密鍵を保持しない」「ユーザーが支出ルール（上限・許可先）を設定・変更できる」「Visa/Mastercard ネットワーク経由のカード決済に対応する」の 3 点で、エージェント決済の「安全アーキテクチャのデファクト」が収束しつつある

2. **規制対応の差が競争上の重要な変数に**: OwlTing が米国 40 州 MTL を保有しているのに対し、Exodus XO Cash の米国 MTL 状況は不明確。コンプライアンスを重視する金融機関・PSP は規制対応済みインフラを優先的に採用するため、MTL・AML・OFAC 対応の有無が今後の差別化要因になる

3. **Monavate が「AI エージェント向けカード発行基盤」のキーポジションを占める**: MoonAgents Card（MoonPay × Monavate）と Exodus XO Cash（Exodus × MoonPay × Monavate）の両製品が Monavate の Mastercard プリンシパルメンバー資格を基盤としており、MoonPay と Monavate が AI エージェント向けデビットカード発行の主要インフラとして台頭している

4. **コンシューマー向け UX と開発者向け API の二極化**: AWS AgentCore・Pay.sh・Exodus AgentKit が開発者向け SDK/API 中心である一方、OwlPay Agent Wallet は「技術設定不要でエージェントがウォレット操作を習得する」というエンドユーザー向け UX を重視する。この二極化はエージェント決済インフラが企業向け（B2B）とコンシューマー向け（B2C）の両軸で同時に整備されていることを示す

## 共通して見える変化

- **既存の規制済みカードインフラの再利用**: MoonPay・Monavate・Visa Direct など、すでに規制対応済みの決済インフラを活用して AI エージェント向け製品を素早く市場投入するアプローチが主流になっている。これはグリーンフィールドでの構築より速く、規制リスクも低い
- **「ユーザーが支出ルールを設定する」制御モデルの標準化**: AWS AgentCore の「支出上限をインフラ層で強制」、Exodus XO Cash の「ユーザーが設定するルールをカード決済にも適用」、OwlTing の「ユーザー授権範囲内で自律実行」──各製品が異なる実装で同一の設計思想（ユーザー制御型支出制限）を採用している
