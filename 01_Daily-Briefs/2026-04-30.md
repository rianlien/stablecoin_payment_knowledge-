# Daily Brief: 2026-04-30

## 今日の要点

- Stripe Sessions 2026（2026-04-30）で、ステーブルコイン決済の受け入れ市場を 32 市場追加し、Bridge の対応チェーン（Tempo・Plasma・Celo・Sui）・ステーブルコイン（USDG・CASH・USDSui・USDCBL）を大幅拡充。USDT は Q2 プレビュー、BTC・SOL・ETH は Q4 プレビューとロードマップを公開。Stripe Docs でも Link 経由のステーブルコイン受け取り仕様（米国・USD 建て限定・チャージバック非対応）が公開された
- Machine Payments Protocol（MPP）を発表。AI エージェントがマイクロ・定期・オンデマンド決済をプログラムで実行できる Stripe 独自のプロトコルで、Link エージェントウォレット・Shared Payment Tokens（ステーブルコイン＋法定＋BNPL を統合）・Privy CLI ウォレットプロビジョニングと組み合わせてエージェント決済のフルスタックを形成
- Radar に「正規 AI エージェント vs 不正アクター」識別機能をプレビュー。エージェント決済の普及に伴うボット悪用リスクへの対応を同時に組み込んだ

## 今日の重要論点

1. **Stripe が「ステーブルコイン清算レイヤー」へ転換した**: Bridge による多チェーン・多通貨対応（Tempo・Celo・Sui・USDG・CASH 等）は、Stripe を特定ステーブルコインに依存しない汎用清算インフラとして位置付ける。Circle CPN との差別化軸が「エコシステムの閉鎖性（Circle）vs 多通貨・多チェーン開放性（Stripe/Bridge）」に移りつつある。

2. **MPP と x402 の並走：エージェント決済標準争いが本格化**: Stripe は x402 Foundation の創設メンバーでもありながら、独自の MPP を発表した。オープン標準（x402・ERC-8183）と Stripe 独自標準（MPP）の関係が整理されていない。Stripe エコシステムに乗るか、オープン標準を選ぶかは、エージェント決済プロダクト設計の核心的な選択になる。

3. **Link ステーブルコイン受け取りの「制約の明確化」が導入判断を促す**: 米国・USD・$1〜$10,000・チャージバック非対応という制約条件が Docs で明示されたことで、「対象ユースケースかどうか」の判断が容易になった。日本・EU 事業者は当面対象外だが、32 市場拡大ロードマップ上での解禁時期の見極めが重要。

## 共通して見える変化

- 今週の Stripe Sessions 発表は、先週の Western Union USDPT・Banking Circle MiCA・Veem-Bridge に続く「大手プレイヤーによるステーブルコイン本格統合」の流れを強化した。「ステーブルコインを使うかどうか」から「どのインフラ（Stripe/Bridge か Circle CPN か x402 か）を選ぶか」という競争軸への移行が明確
- エージェント決済では MPP（Stripe）・x402（Linux Foundation）・ERC-8183（Ethereum）の 3 系統が競合し始めており、標準化競争が「プロトコル vs インフラ優位」の構図に入った。Coinbase（Agentic.market）が x402 を選択している一方、Stripe は自社インフラへの引き込みを図っている

  - Bridge の COP・GBP 追加が自社の対象市場と重なる場合、Bridge API 優先評価を検討
