    Weekly Digest: 2026-W18¶

    今週の要点¶

    • アジェンティック決済インフラが「プロトコル宣言期」を脱し、商用稼働と標準競争フェーズへ移行した。x402（Coinbase Agentic.market）・Stripe MPP・Circle Nanopayments・Google AP2（FIDO）の4系統が同週に並走する状態となり、カードレール型（Oobit・Mastercard Agent Pay）を加えると実質5アーキテクチャが競合している
    • Stripe Sessions 2026（4/30）がその中核イベントとなり、ステーブルコイン決済32市場拡大・Bridge多チェーン強化・MPP発表が重なった。同日・翌日のMeta USDC支払い開始・Visa 9チェーン清算 年率70億ドル突破との合流で、Stripe→Visa→Meta という決済スタックが実稼働したことが確認された
    • GENIUS Act の三本柱（FDIC発行基準・FinCEN/OFAC AML・財務省州規制フレームワーク）が出揃い、規制の制度設計が収束フェーズへ。英国も単一決済フレームワークを発表し、規制の「方針決定」から「詳細設計」への移行が米英双方で同時進行している

    今週の重要テーマ¶

    1. エージェント決済インフラの「標準争い」が本格化：x402（Linux Foundation / Coinbase）・Stripe MPP・Circle Nanopayments・Google AP2（FIDO）の 4 系統が商用フェーズで並走。AP2 と x402 は補完スタックとして統合される方向にある一方、Stripe MPP はクローズドな傾向が強く、Mastercard は OpenAI とのクローズド協業を進めている。標準化 vs クローズドエコシステムの分岐が今後の市場構造を左右する

    2. Stripe が「汎用ステーブルコイン清算インフラ」に転換：Meta（支払い）・Visa（清算）・Nium（190 か国フィアット払い出し）が Stripe/Bridge スタックを採用する構造が実稼働した。Circle CPN との競争軸は「多通貨・多チェーン開放性」に移った

    3. レガシー金融の参入が「API ユーザー」から「インフラ参加者」へ：Western Union（DAN）・Kyriba（TAI）・OnePay（Tempo バリデータ）が単なる API 接続を超え、インフラ運営・ガバナンスにコミットし始めた。ステーブルコインネットワークの設計に既存金融の論理が入り込む

    4. USDC のデファクト化とUSDT の法人エージェント向け先行：Meta・Visa・Nium はすべて USDC を採用。一方 Oobit Agent Cards は USDT×Visa で先行商用化し、GENIUS Act PPSI 規制外というポジションを活用。USDC（規制準拠・企業 TMS）vs USDT（法人エージェント日常支出）の棲み分けが浮上

    5. 規制が「方針決定」から「詳細設計」へ：GENIUS Act は州規制 NPRM・AML NPRM・FDIC 発行基準の三本柱が出揃い、2026 年中の最終規則化へ。英国は FCA 認可受付を 9 月に控えバッキング資産要件の最終設計が焦点に

    何が変わりつつあるか¶

    • 決済実運用:
    • 「ステーブルコインを使うインフラを作る」競争が、暗号ネイティブ企業から規制済みの大手金融機関・送金会社・カードネットワークへ主役が移っている

    • プラットフォーム（Meta）→ PSP（Stripe）→ ブロックチェーン（Tempo/Solana）→ カードネットワーク（Visa）という決済スタックが実際の決済フローとして機能し始めた

    • 規制 / リスク:

    • 米国の規制枠組みが「方針決定フェーズ」から「詳細設計フェーズ」へ。連邦 PPSI 義務・州規制基準・利回り可否の三点が固まれば発行体・PSP の実務対応が本格始動する
    • 英国は単一フレームワーク統合の方向性が明確だが、バッキング資産要件が業界との最終調整を残している

    • 分散型ステーブルコイン（DAI）が主流 CEX から排除される方向が加速。規制準拠型（USDC・USDT・USDS）への重力移動

    • プロダクト / 技術:

    • エージェント決済インフラが「プロトコル宣言」から「本番稼働と標準化」フェーズへ移行した週

    • Solana の「決済インフラ」としての地位が固まりつつある（Western Union・Meta・Visa・Shinhan Card が採用）

    • 事業 / 競争:

    • Tempo エコシステムの集積速度が競合チェーン（Polygon・Avalanche 等）を圧倒しており、Stripe が育成した L1 が業界標準ブロックチェーンとして定着しつつある
    • B2B クロスボーダーステーブルコイン決済の実用化事例（WU・Banking Circle・dLocal・Nium）が短期間に集中し、「2026 年が本格普及元年」の構図が固まってきた

    我々への示唆¶

    • いま検討すべきこと:
    • エージェント決済 4 アーキテクチャ（x402・Stripe MPP・Circle Nanopayments・AP2）の比較マトリクスを整理し、自社エージェント決済プロダクト設計における採用候補を絞る。AP2（認証）＋ x402（USDC 決済）の補完スタックがオープン性の観点で有力な選択肢
    • Stripe Link ステーブルコイン仕様の制約（米国・USD 限定・$10,000 上限・チャージバック非対応）が自社プロダクトの対象市場・通貨と合致するか評価
    • 連携先発行体が連邦 PPSI ルートか州規制ルートかを確認し、それぞれの規制義務をリスクマップに反映する

    • 利回り解禁シナリオを前提にした、利回り付きステーブルコインをキャッシュマネジメントに組み込む場合のシステム要件を仮設計する

    • 継続監視すべきこと:

    • CLARITY Act 利回り条項の 5 月マークアップ結果（最重要関門）
    • Western Union USDPT の 5 月ローンチ正式発表と対象国リスト
    • Circle Nanopayments の実取引量と KYT 義務帰属に関する Circle の公式見解
    • Mastercard × OpenAI agent-to-agent 決済の仕様公開タイミング

    • Stripe MPP の仕様公開と x402 との相互運用性が明確になり次第、自社エージェント決済の標準選択を判断

    • 後で深掘りすべきこと:

    • AP2 の Verifiable Intent 標準と GENIUS Act における「エージェント取引の説明責任」要件との整合性
    • Circle Nanopayments における KYT・AML 義務の帰属設計（Circle Gateway が一括報告主体になるか否か）
    • x402 プロトコルにおける KYT・AML 義務の帰属整理（規制上の位置づけが未確定）

    今週の未解決論点¶

    1. エージェント決済の「標準」はどこに収束するか：AP2＋x402（オープンコンソーシアム）・Stripe MPP（インフラ優位型）・Mastercard×OpenAI（クローズド）の三軸が並走している。規制整合（GENIUS Act・MiCA）においてどれが「推薦標準」とされるかが分岐点

    2. 利回り条項の決着：CLARITY Act の Tillis-Alsobrooks 案（パッシブ利回り禁止）に業界 100 社超が反発。5 月マークアップでどう決着するかによって、ステーブルコインの財務管理用途への普及速度が大きく変わる

    3. Circle Nanopayments の KYT 可視性問題：バッチ処理による個別トランザクションの KYT 可視性低下が規制対応上の課題。Circle が AML 義務をどこに帰属させるかの公式見解が出ていない

    4. Solana vs Tempo の共存：Meta・Western Union・Shinhan Card が Solana を直接採用する一方、Stripe エコシステムは Tempo を推進している。長期的にどちらが「メインのステーブルコイン決済チェーン」として定着するか

    5. USDT の法人エージェント用途における規制扱い：Oobit Agent Cards が GENIUS Act PPSI 規制外の USDT で先行商用化したことで、規制準拠のステーブルコイン採用を遅らせる誘因が生まれた可能性がある
